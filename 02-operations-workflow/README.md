# 02 运营全流程（PMP 五过程组）

将 PMP（Project Management Professional）五过程组框架映射到电商运营全链路。

```
启动(Phase 0) → 规划(Phase 1) → 执行(Phase 2) → 监控(Phase 3) → 收尾(Phase 4)
```

## 流程速览

| 过程组 | 核心动作 | 产出 |
|--------|---------|------|
| Phase 0：启动 | 立项章程 + 干系人识别 | 项目目标书、干系人登记册 |
| Phase 1：规划 | 市场/竞品分析 + 战略矩阵 + 热词/视觉/定价规划 + 风险登记册 | 完整运营规划文档 |
| Phase 2：执行 | 视觉制作 + 上架运营 + 付费投放 + 达人合作 | 上架商品、投放计划 |
| Phase 3：监控 | 漏斗日报/周报 + EVM绩效 + 竞品监控 | 异常预警 + 变更请求 |
| Phase 4：收尾 | 阶段复盘 + SOP迭代 | 经验教训登记册 |

## 关键原则

> **分析先行，战略从分析中生长。**
>
> 商业分析（市场 + 竞品）→ 发现机会点/差距/空白 → 战略矩阵 → 执行 → 监控 → 收尾

> **严禁**：先拍脑袋定战略，再找数据凑理由。
