# 电商全链路运营知识库

基于 PMP 五过程组框架的货架电商运营方法论，覆盖从市场分析、竞品拆解到定价、视觉、投放、监控的完整运营链路。

## 使用说明

- **纯浏览**：按下方目录选择感兴趣的模块
- **实操**：配合 `06-模板工具` 中的模板，填充真实数据
- **AI 辅助**：加载 `skill/` 目录作为 Claude Code skill，获得交互式运营诊断

## 目录

| 模块 | 内容 | 适合谁 |
|------|------|--------|
| [01 赚钱逻辑](./01-profit-logic/) | 电商利润公式、四大杠杆、现金流原则 | 运营负责人、老板 |
| [02 运营全流程](./02-operations-workflow/) | PMP五过程组（启动→规划→执行→监控→收尾） | 项目经理、运营主管 |
| [03 分析方法](./03-analysis-methods/) | 销售漏斗、竞品拆解、热词分析、视觉分析、全媒体分发 | 运营专员、分析师 |
| [04 定价体系](./04-pricing/) | 五价联动模型、定价决策矩阵、毛利测算 | 运营、供应链 |
| [05 战略矩阵](./05-strategy/) | 产品矩阵、店铺矩阵、品牌矩阵 | 运营负责人、品牌经理 |
| [06 模板工具](./06-templates/) | WBS、RACI、漏斗报表、热词分析表、定价计算器、上架检查单 | 全团队 |

## 快速开始

**从0到1搭建电商运营体系**：
1. 打开 [上架检查清单](./06-templates/launch-checklist.md)，建立全局视图
2. 按 [运营全流程 Phase 0](./02-operations-workflow/phase0-initiating.md) 完成立项
3. 使用 [竞品分析框架](./03-analysis-methods/competitor-analysis.md) 摸清市场
4. 制定 [产品矩阵](./05-strategy/product-matrix.md) 和 [定价方案](./04-pricing/five-price-model.md)
5. 搭建 [销售漏斗监控](./03-analysis-methods/sales-funnel.md)

**排查具体问题**：
- CTR 低 → [销售漏斗分析 — 诊断追问链](./03-analysis-methods/sales-funnel.md#4-漏斗诊断)
- 转化率低 → [四大杠杆 — 杠杆二：转化率](./01-profit-logic/four-levers.md#杠杆二转化率)
- 定价没把握 → [五价联动模型](./04-pricing/five-price-model.md)
- 竞品打不过 → [竞品三维拆解](./03-analysis-methods/competitor-analysis.md)

## 核心原则

> **分析先行，战略从分析中生长** — 严禁拍脑袋定战略再找数据凑理由。

> **现金流是一切策略的过滤器** — 任何方案必须先过现金流测试。
