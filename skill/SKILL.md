# SKILL.md — 电商全链路运营工作流

## 0. 身份定义

你不是助手。你是电商运营的**认知合伙人**。

基于PMP五过程组框架，结合中国传统电商实操经验，提供从战略规划到执行落地的全链路支持。

---

## 核心哲学（v3.0 新增）

### 分析先行，战略从分析中生长

```
商业分析（市场 + 竞品）→ 发现机会点/差距/空白 → 战略矩阵（品牌/店铺/品类/产品）
                                                                                ↓
                                              执行（热词/视觉/定价/上架/投放）←←←
                                                                                ↓
                                              监控（漏斗/品类健康/库存健康/EVM/策略调整）←←←
                                                                                ↓
                                              收尾（阶段复盘/SOP迭代）←←←←←←←←←←←
```

**严禁**：先拍脑袋定战略，再找数据凑理由。

### 现金流是一切策略的过滤器

任何方案必须过现金流测试：
- 定价策略毛利率 ≥ 平台扣点 + 物流 + 退货率
- 付费投放 CAC < 客单价 × 毛利率
- 引流款目的：带动利润款，不是利润本身

---
## 电商赚钱底层公式

### 表层公式（常见但不完整）

```
GMV = 流量 × 转化率 × 客单价
```

### 完整利润公式

```
利润 = (客单价 × 毛利率 - CAC) × 付费用户数 × 复购次数
```

> 核心原则：不是做大分子，而是控制分母。流量是买来的，CAC控得住，模型才成立。

---

### 四大杠杆方向

#### 🔵 杠杆一：流量（做精不做大）

| 维度 | 关键动作 | 核心指标 |
|------|---------|---------|
| 搜索流量 | 标题词根优化、直通车精准词 | CTR 目标 ≥ 行业均值 |
| 内容流量 | 小红书/抖音种草、达人矩阵 | 播放→主页跳转率 |
| 私域流量 | 老客激活、社群运营 | 复购率、开口率 |
| 活动流量 | 大促报名、平台资源位 | ROI ≥ 2.5 |

> ⚠️ 流量质量 > 流量数量。精准UV 1万比泛流量 10万更赚钱。

**诊断追问**：流量不足时，先问「是搜索词曝光不够，还是CTR太低？」

---

#### 🟢 杠杆二：转化率（说服力系统）

| 影响层 | 核心变量 | 优化动作 |
|--------|---------|---------|
| 主图层 | 第1张白底图点击率 | A/B测试主图，目标CTR提升30% |
| 详情层 | 痛点→卖点→信任背书链 | 首屏3秒留存，增加买家秀 |
| 价格层 | 锚点感知、促销感知 | 划线价×1.3-1.5，限时优惠 |
| 信任层 | DSR评分、评价数量 | 维护DSR ≥ 4.8，差评快速响应 |

> ⚠️ 客单价越高，决策周期越长，转化率越低。高客单品类转化率2-3%已属正常，不要对标快消品。

**诊断追问**：转化率低时，先问「是访客质量低，还是详情页说服力不足？」

---

#### 🟡 杠杆三：客单价（结构性提升）

| 策略 | 实现方式 | 适用场景 |
|------|---------|---------|
| 关联销售 | 主品+配件+服务套装 | 提升连带率 |
| 品类升级 | 引导选择更高规格/套餐 | 详情页对比表 |
| 会员价值锚点 | 会员专属价 vs 日常价 | 私域转化 |
| 凑单机制 | 满减促凑单 | 大促期间 |

> ⚠️ 客单价和转化率是跷跷板，提客单不能以牺牲转化率为代价，需同步强化说服力。

**诊断追问**：客单价偏低时，先问「是产品矩阵缺少高价款，还是用户没被引导升级？」

---

#### 🔴 杠杆四：复购（LTV资产）

```
LTV（用户生命周期价值）= 客单价 × 复购次数
```

| 复购驱动力 | 动作 | 说明 |
|-----------|------|------|
| 耗材/周期提醒 | 到期前30天触达老客 | 私域/短信/push |
| 会员体系 | 积分、等级、专属折扣 | 老客激活核心手段 |
| 内容留存 | 品类知识、使用技巧内容 | 公众号/社群持续输出 |
| 服务绑定 | 增值服务锁定复购 | 提升转换竞品成本 |

> ⚠️ 私域复购比拉新便宜10倍以上。存量用户是成本最低的增长来源。

**诊断追问**：复购率低时，先问「是没有触达机制，还是触达了但没有转化？」

---

### 四杠杆优先级原则

```
资源有限时，优先顺序：

① 复购（成本最低，存量挖掘）
② 转化率（流量不变，产出翻倍）
③ 客单价（结构优化，毛利提升）
④ 流量（成本最高，放在最后加码）
```

## Phase 0.5：经营架构定义（新增 v2.0）

> ⚠️ **前置要求**：在启动任何电商项目前，必须先完成经营架构的四维坐标定义。跳过此步骤将导致后续策略与执行偏离实际经营结构。

### 0.5.0 四维坐标定义框架

每位经营者在启动项目前，必须回答以下四个维度：

```
品牌维度：这个项目属于哪个品牌？品牌定位是什么？
         → 各品牌定位不同，对应的人群、价格带、内容策略均不同

店铺维度：在哪个平台的哪家店？店铺类型是什么（旗舰/专卖/折扣）？
         → 不同店铺类型的平台资源、用户预期、运营重点不同

产品维度：这个SKU在矩阵中是什么角色？
         → 引流款 / 利润款 / 品牌锚点款 / 防御款（四象限）

平台维度：这个平台的核心流量逻辑是什么？
         → 各平台流量分配机制不同，运营策略完全不同
```

### 0.5.1 产品矩阵角色定义（四象限）

| 矩阵角色 | 定价策略 | 流量策略 | 毛利预期 | 考核指标 |
|---------|---------|---------|---------|---------|
| **引流款** | 低于竞品5-10% | 付费投放+大促报名 | 允许微亏 | 点击量/加购量 |
| **利润款** | 市场均价区间 | 搜索自然流+精准投放 | 目标毛利率30%+ | ROI/净利润 |
| **品牌锚点款** | 高于市场均价30-50% | 内容种草+品牌词投放 | 高毛利 | 客单价/品牌搜索量 |
| **防御款** | 跟随竞品定价 | 最小化投入 | 保本即可 | 市场份额守住 |

**关键原则**：
- 同一个店铺四类产品都要有，缺一不可
- 引流款不能无限亏损，设置止损线（如：引流款月亏损上限=利润款月净利润的30%）
- 品牌锚点款不考核销量，只考核品牌感知

### 0.5.2 多平台流量逻辑对照表

| 平台 | 核心流量逻辑 | 用户心智 | 主推产品类型 | 内容形式 |
|------|-----------|---------|-----------|---------|
| 淘宝/天猫 | 搜索意图驱动 | "我知道我要买什么" | 利润款/品牌锚点款 | 图文详情+直播 |
| 京东 | 品质信任驱动 | "我要买正品/快递快" | 品牌锚点款/利润款 | 图文为主 |
| 拼多多 | 价格敏感驱动 | "我要最便宜的" | 引流款/防御款 | 活动报名为主 |
| 抖音 | 内容兴趣驱动 | "这个看着不错" | 引流款/品牌锚点款 | 短视频+直播 |

> ⚠️ 同一产品在不同平台承担不同角色。例如：净水器A在淘宝是利润款，在拼多多可能是防御款。

### 0.5.3 跨平台选品差异策略

| 策略维度 | 高客单平台（淘宝/京东） | 低客单平台（拼多多） | 内容平台（抖音） |
|---------|------------------|-----------------|-------------|
| SKU规格 | 展示最高规格引导升级 | 展示最低规格降低心理门槛 | 展示最具视觉冲击力规格 |
| 主图逻辑 | 功能对比+信任背书 | 价格锚点+性价比感知 | 使用场景+情绪价值 |
| 标题词根 | 品类词+功能词+品牌词 | 价格词+活动词+品类词 | 场景词+痛点词+效果词 |
| 促销方式 | 会员折扣+关联推荐 | 满减+拼单+红包 | 限时秒杀+达人口播 |

### 0.5.4 品牌层级与店铺布局矩阵（多品脾×多店铺）

```
示例（供参考，根据实际经营结构调整）：

品牌A（高端定位）
  ├── 天猫旗舰店 → 品牌锚点款+利润款为主
  ├── 京东旗舰店 → 品牌锚点款+利润款为主
  └── 抖音品牌店 → 内容种草+品牌锚点款

品牌B（中端定位）
  ├── 天猫旗舰店 → 利润款+引流款
  ├── 拼多多旗舰店 → 引流款+防御款
  └── 抖音矩阵号 → 引流款+内容带货

品牌C（性价比定位）
  ├── 拼多多旗舰店 → 引流款为主
  └── 淘宝C店 → 防御款+清仓
```


### 0.5.5 用户画像与决策路径（v4.0 新增）

> ⚠️ 同样的产品，不同的用户，决策路径完全不同。不理解用户画像，所有"优化"都是盲人摸象。

#### 用户决策风格分类

| 决策风格 | 特征 | 详情页策略 | 主图策略 | 促销策略 |
|---------|------|----------|---------|---------|
| **理性对比型** | 比参数、比价格、看测评 | 参数对比表 + 竞品对比图 | 卖点罗列清晰 | 强调性价比 |
| **感性冲动型** | 看场景、看颜值、看氛围 | 场景代入 + 情绪文案 | 高颜值场景图 | 限时限量刺激 |
| **品牌信任型** | 认品牌、看资质、看口碑 | 品牌故事 + 资质展示 | 品牌调性统一 | 会员专属优惠 |
| **价格敏感型** | 比价、等促销、算性价比 | 价格锚点 + 到手价计算 | 促销标签突出 | 满减/拼单/红包 |
| **便捷效率型** | 看物流、看售后、看库存 | 物流时效 + 售后保障 | 现货/次日达标识 | 免运费/急速达 |

#### 购买场景识别

| 使用场景 | SKU策略 | 包装策略 | 话术侧重 |
|---------|--------|---------|---------|
| **自用** | 标准规格为主 | 环保简约 | 强调实用性和性价比 |
| **送礼** | 礼盒装/高端规格 | 精美礼盒+手提袋 | 强调档次和仪式感 |
| **商用/批量** | 大包装/批发规格 | 工业包装 | 强调稳定性和批量优惠 |

### 1.2 真实毛利率管理（盲区C1 新增 v2.0）

> ⚠️ 定价不等于利润。很多运营只看「售价-进货价」的账面毛利，忽略了平台扣点、运费、包装、退货等隐性成本，导致账面盈利实际亏损。

#### 毛利率三层次计算

```
层次一：账面毛利率（最常见的坑）
  = (售价 - 进货价) / 售价 × 100%

层次二：平台实收毛利率
  = (售价 - 进货价 - 平台扣点 - 运费 - 包装) / 售价 × 100%

层次三：真实净利润率（扣除退货后）
  = (售价×(1-退货率) - 进货价 - 平台扣点 - 运费 - 包装 - CAC) / 售价 × 100%
```

#### 各平台成本结构参考

| 成本项 | 淘宝/天猫 | 京东 | 拼多多 | 抖音 |
|-------|---------|------|-------|------|
| 平台扣点 | 2-5% | 5-8% | 0.6-2% | 2-5% |
| 运费均摊 | 3-8元/单 | 5-10元/单 | 2-5元/单 | 3-8元/单 |
| 包装成本 | 2-5元/单 | 3-8元/单 | 1-3元/单 | 2-5元/单 |
| 退货率参考 | 5-15% | 3-10% | 10-25% | 15-35% |

#### 毛利诊断速查

| 账面毛利率 | 平台实收毛利率 | 判断 | 行动 |
|----------|-------------|------|------|
| ≥50% | ≥35% | ✅ 健康 | 可放量投放 |
| 30-50% | 15-35% | ⚠️ 警戒 | 审视隐性成本 |
| ≤30% | ≤15% | 🚨 危险 | 不整改不能投流 |

> 核心原则：**投放前必须先算三层毛利。账面毛利率低于30%的品类，不建议做付费投放。**

### 1.3 库存与现金流管理（盲区C2 新增 v2.0）

> ⚠️ 电商公司死因排名第一不是没流量，是现金流断裂。爆款也可能是催命符——卖得越多，压货越多，回款越慢。

#### 库存健康度三色模型

```
🟢 健康库存（周转天数 15-30天）
   → 正常销售，按周补货

🟡 预警库存（周转天数 30-60天）
   → 启动促销/跨店调拨/达人清仓

🔴 危险库存（周转天数 >60天）
   → 立即折价清仓，不计毛利，只求回款
```

#### 现金流三条生命线

| 生命线 | 含义 | 警戒阈值 |
|-------|------|---------|
| 红线（死亡线） | 可用现金 = 0，货款未回 | 7天内若无回款即断流 |
| 黄线（警戒线） | 可用现金 < 未来30天应付 | 停止所有新采购 |
| 绿线（安全线） | 可用现金 > 未来60天应付 | 可正常运营 |

#### 库存-现金流联动管理

```
备货金额上限 = 可用现金 × 40%（单次备货不超现金流40%）

安全库存量 = 日均销量 × 采购周期天数 × 1.5（安全系数）

清仓触发点：
  - 库存周转 >45天 或 可用现金 < 黄线
  - 立即启动： ①停止进货 ②跨店调拨 ③达人清仓 ④平台活动清仓
```

> 核心原则：**宁可断货，不可断流。缺货损失的是销量，断流损失的是公司。**

#### 库存指标补充（v3.0 扩展）

| 指标 | 计算方式 | 预警阈值 | 触发动作 |
|------|---------|---------|---------|
| **库存周转天数** | 平均库存 / 日均销量 | > 45天 | 促销清仓 |
| **缺货预警** | 可售库存 < 日均销量 x 补货周期 x 1.5 | 连续2天 | 紧急补货 |
| **高库存预警** | 库存周转天数 > 60天 | TOP5 SKU | 降价/捆绑销售 |
| **滞销品识别** | 30天销量 <= 3件 | 占比 > 20% | 下架/替换 |
| **售罄率** | 已售数量 / 上架数量 | > 90%需补货，< 30%需促销 | 按品类判断 |
| **退货率** | 退货订单 / 总订单 | > 10%需排查产品质量 | 改进描述/质检 |

#### 库存优化决策树

`
库存周转天数 > 45天？
  +- 是 -> 是否缺货？
  |      +- 是 -> 紧急补货（安全库存 x 2）
  |      +- 否 -> 滞销分析
  |             +- 30天销量 <= 3件 -> 下架/清仓
  |             +- 30天销量 4-10件 -> 促销（满减/捆绑）
  |             +- 30天销量 > 10件 -> 正常监控
  +- 否 -> 库存健康，维持当前策略
`

### 1.7 定价策略

| 价格类型 | 用途 | 策略逻辑 |
|---------|------|---------|
| **对手价** | 卡位参考 | 截流竞品流量，通常低于竞品5-10% |
| **标价** | 标注划线价 | 制造锚点感知，激发占便宜心理 |
| **日常价** | 日常销售价 | 稳定转化，覆盖运营成本 |
| **活动价** | 平台大促（618/双11） | 冲量保级，毛利可降至盈亏平衡点 |
| **批发价** | 分销/达人带货 | 批量起量，通常为日常价×0.6-0.8 |

```
毛利率 = (日常价 - 产品成本) / 日常价
活动价毛利率 ≥ 平台扣点(5-8%) + 物流(3-5%) + 退货率(5-15%)
CAC(单客成本) < 客单价 × 毛利率
```

#### 定价模型选择指南（v3.0 扩展）

| 模型 | 计算方式 | 适用场景 | 风险 |
|------|---------|---------|------|
| **成本加成** | 成本 × (1 + 目标毛利率) | 标品、成本稳定品类 | 忽视竞争与需求 |
| **渗透定价** | 低于市场均价10-20% | 新店冷启动、抢占份额 | 前期亏损，品牌定位低端化 |
| **撇脂定价** | 高于市场均价30-50% | 品牌锚点款、技术壁垒产品 | 需要强品牌或内容种草支撑 |
| **竞品参考** | 取竞品价格中位数±5% | 常规利润款 | 容易陷入价格战 |
| **五价联动** | 标价/划线价/活动价/会员价/渠道价 | 多店铺多平台运营 | 需要协同管理避免破价 |

> **现金流测试**：任何定价方案必须确保毛利率 ≥ 平台扣点 + 物流 + 退货率，否则无法支撑付费投放。

### 1.10 多平台协同运营（盲区C6 + 模块A3/A4扩展 新增 v2.0）

> ⚠️ 多平台不是把同一套东西复制粘贴到所有平台。每个平台的流量逻辑、用户心智、内容形式完全不同，需要差异化的协同策略。

#### 多平台资源分配原则

| 品牌/店铺类型 | 主力平台 | 投入占比 | 运营重点 |
|------------|---------|---------|---------|
| 品牌锚点款 | 天猫+京东 | 60%预算 | 品牌词排名+内容种草+会员体系 |
| 利润款 | 天猫+京东+抖音 | 30%预算 | 搜索排名+精准投放+达人合作 |
| 引流款 | 拼多多+抖音 | 10%预算 | 活动报名+直播间引流 |
| 防御款 | 拼多多 | 最低预算 | 跟随定价+长尾自然流 |

#### 协同运营三同步

```
同步一：价格同步
  - 同品牌同SKU跨平台价差控制在5%以内
  - 活动价格提前3天同步至所有平台
  - 避免拼多多破价影响天猫/京东价格体系

同步二：库存同步
  - 共用库存池→实时库存同步
  - 任一平台库存低于安全线→全平台预警
  - 爆款单品设置全平台总库存上限

同步三：内容同步
  - 主图/详情核心卖点统一（品牌一致性）
  - 各平台根据流量逻辑做形式适配（不改变核心卖点）
  - 达人内容在不同平台二次分发
```

#### 多平台决策矩阵

当资源有限时使用此矩阵判断优先级：

| 问题 | 回答 | 行动 |
|------|------|------|
| 该品牌主力平台是哪家？ | → 先保主力平台产出 |
| 新品首发在哪个平台？ | → 首发平台先投入，跑通再复制 |
| 竞品在哪个平台最弱？ | → 优先切入竞品弱势平台 |
| 哪个平台ROI最高？ | → 资金优先向高ROI平台倾斜 |

> 核心原则：**先深耕一个平台跑通模型，再复制到其他平台。不要同时启动所有平台，资源会被稀释。**

### 1.11 GMV预估与备货预算推导（v3.0 新增）

> ⚠️ **每个数字必须有来源和公式，不允许直接给结论。**

#### 推导链路

```
竞品数据（30天销量分布）
    ↓
销量预估（日均销量 × 30天）
    ↓
GMV预估（销量 × 定价）
    ↓
备货量推导（日均销量 × 补货周期 + 安全库存）
    ↓
推广预算推导（GMV × 推广占比）
```

#### 销量预估推导

```
公式：月销量预估 = 日均销量 × 30天
日均销量来源：从竞品数据中，目标定价区间的中位数月销量 ÷ 30天
```

**新店/新链接冷启动三阶段目标**：

| 阶段 | 目标 | 说明 |
|------|------|------|
| 第一阶段 | 达到平均线 | 尽快达到竞品中位数销量 |
| 第二阶段 | 搜索页前48位 | 稳定日销，进入第一页 |
| 第三阶段 | 前10位 | 冲击头部排名 |

**空白带处理**：若目标定价落入空白带，取相邻区间插值估算：
```
空白带定价 → 取相邻低价区间中位数 × 0.3~0.5（保守系数）
```

#### GMV预估推导

```
公式：月GMV = 月销量 × 单价
日GMV = 日均销量 × 单价
```

#### 备货量推导

```
公式：备货量 = 日均销量 × 补货周期 + 安全库存
```

| 参数 | 确定方式 |
|------|---------|
| 日均销量 | 竞品数据中位数 ÷ 30 |
| 补货周期 | 供应链实际（自提/物流/供应商交期） |
| 安全库存 | 断货容忍度 × 日均销量（冷启动期0.5~1天） |

#### 推广预算推导

```
公式：月推广预算 = 月GMV × 推广占比
```

| 阶段 | 推广占比 | 说明 |
|------|---------|------|
| 冷启动期 | **GMV × 25%** | 流量培育，以量换排名 |
| 日常运营 | **GMV × 20%** | 稳定流量，维持排名 |

#### 盈亏分析（分阶段）

> **冷启动期不考虑盈亏，日常期才做盈亏分析。**

| 收入/成本项 | 说明 |
|------------|------|
| 月GMV | 100% |
| - 货品成本 | 产品成本率 |
| - 平台扣点 | 各平台扣点率 |
| - 推广费用 | 20-25% |
| - 包装+运费 | 实际核算 |
| = 月净毛利 | |
| - 月固定成本 | 包装/仓储/人工/平台费 |
| **= 月净利润** | |

### 1.12 搜索曝光优化（v4.0 新增）

> ⚠️ 搜索是货架电商最大的流量入口。标题优化只是第一步，曝光→点击的转化还需要理解搜索排序逻辑和用户行为。

#### 搜索排序算法权重拆解

| 权重因子 | 影响范围 | 优化方向 |
|---------|---------|---------|
| **关键词相关性**（30-40%） | 标题与搜索词的匹配度 | 标题词根覆盖+属性词精准填充 |
| **销量权重**（20-30%） | 近30天确认收货销量 | 冷启动期用促销冲量破零 |
| **转化率**（15-20%） | 搜索曝光→点击→成交的转化链 | 主图CTR优化+详情页说服力 |
| **DSR评分**（10-15%） | 描述/服务/物流三维评分 | 维持DSR≥4.8（见3.5） |
| **店铺权重**（5-10%） | 店铺层级/活跃度/违规记录 | 持续上新+避免违规扣分 |

#### 长尾词挖掘策略

| 词类 | 来源 | 特征 | 策略 |
|------|------|------|------|
| **搜索联想词** | 搜索框下拉 | 高频搜索、竞争激烈 | 精准匹配+高出价抢排名 |
| **长尾属性词** | 竞品标题反查 | 搜索量小但转化率高 | 广泛匹配+低出价覆盖 |
| **场景词** | 问大家/评价 | 反映真实使用场景 | 融入标题和详情页内容 |
| **竞品品牌词** | 竞品品牌+品类 | 截流竞品流量 | 投放竞品词（合规范围内） |

#### 筛选条件优化策略

用户在搜索结果页常用筛选条件缩小范围。占据筛选结果的核心是**在这些维度上做到极致**：

| 筛选维度 | 用户常用筛选 | 优化目标 | 实现方式 |
|---------|------------|---------|---------|
| **价格区间** | 按预算筛选 | 定价落在主流筛选区间 | 根据竞品价格带分布定目标价位 |
| **发货地** | 按距离筛选 | 多仓布局覆盖核心消费区 | 云仓/分仓策略 |
| **品牌** | 按品牌筛选 | 品牌词搜索量 | 品牌种草+品牌词投放 |
| **好评率** | ≥4.8分 | 维持高DSR | 售后管控+主动好评引导 |

#### 运费策略设计

> 运费是搜索结果页最显眼的信号之一。「包邮」与「不包邮」的点击率差异可达30%+。

| 策略 | 做法 | 适用场景 | 注意事项 |
|------|------|---------|---------|
| **全店包邮** | 所有商品免运费 | 高毛利/品牌锚点款 | 运费成本计入定价 |
| **门槛包邮** | 满X元包邮 | 低客单/引流款 | 门槛≈客单价×1.2，刺激凑单 |
| **单品包邮** | 爆款包邮，长尾不包 | 利润款+引流款组合 | 爆款引流，长尾赚运费 |
| **区域包邮** | 仅包邮核心消费区域 | 仓储布局有限时 | 非包邮区需明确标注 |

#### 销量数字信号设计

> 冷启动期：0销量是最差信号。用户天然信任"别人买过的"。

| 阶段 | 破零策略 | 节奏 |
|------|---------|------|
| **Day 1-3** | 设置促销价+首单优惠+私域老客内购 | 快速积累10-20单 |
| **Day 4-7** | 加大直通车投放+关联推荐+直播间引流 | 日均出单稳定 |
| **Day 8-14** | 评价引导+买家秀积累 | 同时积累销量和信任信号 |
| **Day 15+** | 根据数据调整：定价/主图/投放预算 | 进入日常运营 |

### 难题一：产品基础资料

#### SOP — 产品信息整理标准流程

```
Step 1 【收集原始资料】
  □ 产品名称（含型号/规格）
  □ 产品图片（白底图/场景图/细节图/包装图）
  □ 核心参数（尺寸/重量/材质/功能/认证）
  □ 生产厂家资质（营业执照/产品认证/检测报告）
  □ 竞品参照（2-3款同类竞品的标题/卖点/主图）

Step 2 【提炼核心卖点】
  □ 功能卖点：产品能做什么（客观参数支撑）
  □ 场景卖点：什么人在什么情况下用（用户画像）
  □ 差异卖点：比竞品好在哪（对比维度）
  □ 信任卖点：凭什么相信你（认证/销量/评价/品牌）

Step 3 【整理成结构化产品信息表】
  □ 标准化字段：名称/类目/主关键词/属性值
  □ 卖点优先级排序（P1主卖点 / P2辅助卖点 / P3背书）
  □ 禁用词/违规词排查（各平台规则不同）

Step 4 【输出物料清单】
  □ 产品信息表（Excel或飞书多维表）
  □ 卖点话术库（供标题/详情/客服共用）
  □ 主图素材清单（标注每张图的用途）
```

#### 诊断协议 — 产品资料问题追问链

| 症状 | 第一追问 | 常见根因 |
|------|---------|---------|
| 卖点说不清楚 | 「你觉得这个产品比竞品好在哪里？」 | 没做竞品对比，差异化不明确 |
| 图片不够用 | 「现在有哪几张图？白底图有没有？」 | 供应商只给了样品图，缺场景图/细节图 |
| 不知道写什么参数 | 「这个产品有没有检测报告或认证文件？」 | 资质不全，参数靠猜 |
| 卖点写了但没人买 | 「这个卖点是你觉得重要，还是买家搜索验证过的？」 | 卖点从供给视角出发，不是需求视角 |

### 难题二：上架

#### SOP — 标准上架操作流程

```
Step 1 【标题撰写】
  公式：核心词 + 属性词 + 场景词 + 长尾词
  □ 核心词：品类大词（搜索量最大）
  □ 属性词：规格/材质/功能修饰
  □ 场景词：使用场景/人群
  □ 长尾词：具体需求词/比较词
  □ 字数：淘宝/京东 30字、拼多多 60字、抖音 20字以内
  □ 排查：禁用词/夸大词/违规表达

Step 2 【主图制作】
  □ 第1张：白底主图（搜索结果页展示，正面全貌）
  □ 第2张：场景图（使用场景，生活化）
  □ 第3张：卖点图（核心差异化，简洁有力）
  □ 第4张：细节图（做工/材质/工艺特写）
  □ 第5张：营销图（促销/背书/认证）
  □ 尺寸：800×800px起，建议1:1方图

Step 3 【属性填写】
  □ 类目选择：选最精准的三级类目
  □ 必填属性：100%填完，不留空
  □ 关键属性：品牌/产地/材质/规格/适用人群
  □ 自定义属性：补充竞品没有但买家关心的维度

Step 4 【SKU设置】
  □ 规格名称标准化（颜色/尺寸/套餐）
  □ 价格梯度合理（大规格应有优惠感）
  □ 库存数量填写（不要填0，影响权重）
  □ 主推SKU放第一位

Step 5 【详情页结构】
  痛点引入 → 产品亮相 → 卖点拆解 → 参数规格
  → 使用方法 → 对比竞品 → 买家秀/评价 → 售后保障

Step 6 【上架前检查】
  □ 预览移动端展示效果
  □ 价格/运费设置正确
  □ 物流模板已配置
  □ 售后规则已设置
```

#### 诊断协议 — 上架问题追问链

| 症状 | 第一追问 | 常见根因 |
|------|---------|---------|
| 上架后没有曝光 | 「类目选对了吗？属性有没有漏填？」 | 类目偏差或属性缺失导致收录异常 |
| CTR极低（<1%） | 「第一张主图是白底图还是场景图？」 | 主图不符合搜索结果页视觉规律 |
| 标题被平台降权 | 「标题里有没有用"最""第一""极品"等词？」 | 违规词触发平台审核 |
| SKU设置混乱 | 「主推的那个规格是第一个吗？」 | 主推SKU未置顶，买家默认选错规格 |
| 详情页跳失率高 | 「首屏有没有在3秒内说清楚产品是什么？」 | 首屏信息密度低，买家看不下去 |

### 难题三：投流

#### SOP — 新手投流启动标准流程

```
Step 0 【投流前提条件确认】
  □ 基础转化率 ≥ 1%（没有转化不要先投流）
  □ 主图/详情页已优化完毕
  □ 有库存支撑（至少能发100单）
  □ 客服已就位（投流后咨询量会增加）

Step 1 【选对第一个工具】
  新手建议顺序：
  ① 先开搜索推广（直通车/商品推广）：意图明确，转化效率高
  ② 再开展示推广（引力魔方/智能投放）：扩大曝光
  ③ 最后考虑内容投流（达人/短视频）：需要素材积累

Step 2 【关键词策略】
  □ 起步：先跑系统推荐词 + 手动加10-20个精准词
  □ 匹配方式：先用广泛匹配积累数据，再收紧精确匹配
  □ 否定词：每3天检查搜索词报告，把不相关词加入否定

Step 3 【出价策略】
  □ 初始出价：参考系统建议出价的80-90%
  □ 调价节奏：3天一个调整周期，单次调幅不超过20%
  □ 时段设置：先全天投放，积累数据后再做分时调整

Step 4 【预算设置】
  □ 日预算：初始不超过目标ROI × 日均客单价 × 5单
  □ 止损线：ROI连续3天低于1.5，暂停检查
  □ 放量条件：ROI稳定 ≥ 2.5，逐步提升日预算上限

Step 5 【数据监控节奏】
  每日：看花费/点击/ROI是否异常
  每3天：看搜索词报告，补词/否词
  每周：看人群报告，优化人群溢价
  每月：复盘ROI趋势，调整整体策略
```

#### 诊断协议 — 投流问题追问链

| 症状 | 第一追问 | 常见根因 |
|------|---------|---------|
| 花钱但没转化 | 「投流前自然搜索转化率是多少？」 | 产品本身转化差，投流放大了亏损 |
| CTR很低 | 「用的是哪个工具？关键词匹配方式是什么？」 | 词不精准/主图弱/出价排名太低 |
| ROI持续低于1 | 「客单价多少？毛利率大概多少？」 | 定价结构问题，投流ROI天花板不够高 |
| 预算烧完但单量少 | 「有没有设置分时折扣？什么时间段花钱最多？」 | 非黄金时段烧掉预算，黄金时段反而没钱投 |
| 不知道该不该继续投 | 「已经投了多少天？积累了多少点击量？」 | 数据量不足就下结论，需至少300次点击再判断 |

| **品类日/超品日** | 品牌资质+独家产品 | 极高 | 有独家产品时重点争取 |
| **百亿补贴** | 价格全网最低+正品保障 | 量大利薄 | 清库存或新品冲量 |
| **首页banner/频道** | 平台小二关系+商品竞争力 | 极高 | 维护平台小二关系 |
| **直播排位赛** | 直播间GMV+互动数据 | 中高 | 日常积累直播间权重 |

## 难题四：自动客服

#### SOP — 自动客服配置标准流程

```
Step 1 【梳理高频问题清单】
  从以下来源整理买家最常问的问题：
  □ 问大家（竞品问答区）
  □ 评价关键词（买家真实痛点）
  □ 历史聊天记录（人工客服整理）
  □ 产品详情FAQ模块

  高频问题分类：
  A类（可完全自动回复）：快递/发货时间、退换货政策、颜色/尺寸确认
  B类（自动回复+人工介入）：使用方法、适配性问题、优惠询价
  C类（必须人工）：投诉、差评处理、大额定制

Step 2 【编写话术库】
  每条话术结构：
  □ 触发词（买家可能怎么问）
  □ 自动回复内容（简洁、有温度、带行动引导）
  □ 结尾钩子（引导加购/收藏/下单）

  话术原则：
  □ 不超过3句话
  □ 先答问题，再引导转化
  □ 避免机器感（加语气词、口语化表达）

Step 3 【配置自动回复规则】
  □ 关键词触发：设置同义词/近义词（防漏触）
  □ 欢迎语：店铺进入自动发送（含当前促销信息）
  □ 超时提醒：买家等待超过1分钟自动发送安抚语
  □ 离线设置：非工作时间自动告知预计回复时间

Step 4 【人工介入判断规则】
  以下情况自动转人工：
  □ 买家连续发送3条消息未解决
  □ 出现"投诉""差评""退款""质量问题"等关键词
  □ 订单金额超过设定阈值
  □ 买家明确要求"人工"

Step 5 【持续优化节奏】
  □ 每周：看未命中关键词报告，补充触发词
  □ 每月：统计人工介入率（目标：A类问题自动率 ≥ 80%）
  □ 每季：更新话术（促销节点/新品信息/政策变化）
```

#### 诊断协议 — 自动客服问题追问链

| 症状 | 第一追问 | 常见根因 |
|------|---------|---------|
| 买家不理自动回复 | 「回复内容有几句？有没有直接回答买家的问题？」 | 话术太长/绕弯子，买家看不到答案 |
| 漏单率高 | 「哪个时间段漏单最多？是否有离线自动回复？」 | 非工作时间无人接管，买家等待后流失 |
| 自动回复触发不了 | 「关键词设置了哪些？买家实际是怎么问的？」 | 触发词覆盖不全，买家用了不同表达 |
| 买家投诉客服差 | 「投诉的是自动回复还是人工回复？」 | 自动/人工边界不清，复杂问题没转人工 |
| 咨询转化率低 | 「话术结尾有没有引导下单的动作？」 | 话术只答疑没有转化钩子 |

## 详情页信任体系（v4.0 新增）

> ⚠️ 货架电商的本质是「在不认识的人那里买东西」。每一个信任信号都在回答用户心中的一个问题：「我凭什么相信你？」

### 信任信号金字塔

`
                    ┌──────────────┐
                    │  平台担保层    │ ← 消费者保障/正品险/假一赔十
                    │              │
                 ┌──┴──────────────┴──┐
                 │    社交验证层       │ ← 评价/买家秀/问大家/KOL种草
                 │                    │
              ┌──┴────────────────────┴──┐
              │      品牌权威层           │ ← 资质认证/检测报告/品牌故事
              │                          │
           ┌──┴──────────────────────────┴──┐
           │        交易安全层               │ ← 运费险/7天无理由/保价/极速退款
           │                                │
        ┌──┴────────────────────────────────┴──┐
        │           基础信息层                   │ ← 店铺类型/DSR/销量/发货地
        └───────────────────────────────────────┘
`

### 问大家（问答模块）运营

> **「问大家」是货架电商独有的、最具说服力的信任资产**——它是已购用户之间的真实问答，买家更信任同类消费者而非商家。

#### 问大家价值

| 维度 | 说明 |
|------|------|
| **真实性** | 问题由真实买家提出，回答来自已购用户（非商家可操控） |
| **精准性** | 问题往往是详情页未覆盖的"最后一公里疑虑" |
| **持久性** | 优质问答长期沉淀在详情页，持续影响转化 |
| **SEO价值** | 问答内容可能被搜索引擎收录，带来额外流量 |

#### 问大家运营标准流程

`
Step 1 — 问题预埋（上架前）
  → 根据竞品问大家和自身产品痛点，预设5-10个高频问题
  → 邀请首批购买用户回答（不可自己回答）
  → 问题类型：质量疑虑/使用场景/售后/对比竞品

Step 2 — 日常监控（每周）
  → 检查是否有未回复的新问题（超过24h未回复影响转化）
  → 发现负面问答 → 联系用户解决 → 引导更新回答
  → 发现高频问题 → 同步更新详情页内容

Step 3 — 内容反哺（每月）
  → 提取问大家中的高频关键词 → 优化标题和详情卖点
  → 发现竞品对比问题 → 补充竞品对比图到详情页
  → 提取用户真实使用场景 → 优化场景图策划
`

#### 问大家诊断速查

| 症状 | 诊断 | 行动 |
|------|------|------|
| 问题数为0 | 无引导机制 | 上架前预埋5-10个种子问题 |
| 问题多但无回答 | 无人维护 | 联系已购用户邀请回答 |
| 负面问题占主导 | 产品质量或描述问题 | 先解决产品问题，再邀请满意用户回答 |
| 大量重复问题 | 详情页信息缺失 | 将高频问题的答案写入详情页 |

### 资质与认证展示策略

| 资质类型 | 展示位置 | 适用品类 | 设计要点 |
|---------|---------|---------|---------|
| **检测报告** | 详情页首屏或信任区 | 食品/母婴/电器/化妆品 | 标注关键指标，避免全文堆砌 |
| **获奖证书** | 主图第5张或详情页 | 全品类 | 放在显眼位置，标注获奖年份 |
| **专利证书** | 详情页技术说明区 | 有技术壁垒的品类 | 标注专利号和核心创新点 |
| **品牌授权** | 主图或详情首页 | 代理/经销商品 | 注意授权期限，过期立即更新 |
| **工厂实拍** | 详情页或视频 | 强调源头直供 | 真实场景优于精修图 |

### 包装展示与开箱体验

> 包装是用户收到产品后的第一个物理触点，直接影响好评率和复购意愿。

| 包装层级 | 适用场景 | 成本参考 | 对转化影响 |
|---------|---------|---------|----------|
| **基础层**（防护包装） | 低客单/引流款 | 1-3元 | 减少运输损坏差评 |
| **标准层**（品牌包装） | 利润款/日用品 | 3-8元 | 提升品牌认知和复购 |
| **体验层**（礼品包装） | 品牌锚点款/礼品 | 8-20元 | 大幅提升好评率和社交传播 |
| **惊喜层**（开箱体验） | 高客单/社交属性 | 15-50元 | 自带社交传播属性，用户主动晒图 |

### 价格保价与信任消除

| 保价策略 | 适用场景 | 对转化影响 |
|---------|---------|----------|
| **7天/15天保价** | 日常销售 | 消除"等促销"心理 |
| **大促保价** | 618/双11期间 | 预付定金用户无需担心降价 |
| **买贵退差** | 价格敏感品类 | 直接消除比价顾虑 |
| **不保价说明** | 清仓/限量 | 明确告知"售完即止"，制造稀缺感 |

> 核心原则：**信任不是靠一句话建立的，而是靠每一个细节信号的叠加。一个差评抵消十个好评，一个漏洞抵消一百个努力。**

## 难题五：私域运营（新增 v2.0）

> ⚠️ 私域是复购的核心基础设施，不是可选项。没有私域的电商，每次销售都是从零开始重新获取用户，CAC永远降不下来。

### 私域运营四个阶段

```
Level 1：沉淀基础（必须具备）
  → 包裹卡+客服引导 把已购用户导入企业微信/社群
  → 目标：已购用户沉淀率 ≥ 30%

Level 2：分层触达（1个月内完成）
  → 按购买品类/价格分人群标签
  → 目标：实现差异化内容推送

Level 3：持续激活（日常运营）
  → 定期内容输出（使用技巧/品类知识/新品预告）
  → 目标：月活跃率 ≥ 20%

Level 4：裂变增长（稳定后启动）
  → 老带新/社群裂变/分销机制
  → 目标：私域贡献GMV占比 ≥ 15%
```

### 私域触达矩阵

| 触达方式 | 频率 | 适用内容 | 注意事项 |
|---------|------|---------|---------|
| 企业微信群发 | 每周1-2次 | 新品发布、大促预告 | 不打扰用户，提供退出入口 |
| 社群日常运营 | 每天1-2条 | 使用技巧、行业资讯 | 提供价值，不纯销售 |
| 朋友圈 | 每周3-5条 | 产品故事、用户好评 | 以内容而非广告形式 |
| 公众号 | 每周1-2篇 | 深度内容、选购指南 | 与短视频内容协同 |
| 视频号/小程序 | 持续更新 | 直播切片、产品演示 | 与公域流量互补 |

### 私域引流四大入口

| 入口 | 方法 | 预估转化率 |
|------|------|---------|
| 包裹卡 | 印企业微信二维码+福利钩子 | 5-15% |
| 客服话术 | 成交后引导添加「专属顾问」 | 10-20% |
| 短信/Push | 已购用户复购提醒+私域入口 | 2-5% |
| 直播间引导 | 口播+弹窗引导加粉丝群 | 3-8% |

### 私域复购驱动模型

```
私域用户复购路径：

添加企微 → 收到欢迎语+首次福利 → 进入分组标签
  → 第7天：使用技巧内容触达
  → 第15天：关联品类推荐
  → 第30天：复购专属优惠
  → 第45天：会员升级邀请
  → 持续：定期品类知识+专属活动
```

### 私域诊断速查

| 症状 | 第一追问 | 常见根因 |
|------|---------|---------|
| 沉淀率低 | 「包裹卡和客服话术都落地了吗？」 | 缺乏引导机制或福利不够吸引 |
| 社群死寂 | 「最近一次有价值的内容是什么时候？」 | 内容全是广告，没有用户价值 |
| 复购率不提升 | 「私域用户收到过几次专属优惠？」 | 加了私域但没有差异化对待 |
| 用户大量退群 | 「退群前最后一条消息是什么？」 | 过度营销骚扰 |


### 社交裂变与口碑传播（v4.0 扩展）

> 朋友推荐是所有渠道中转化率最高的——没有之一。私域的终极价值是让用户成为传播节点。

#### 裂变机制设计

| 裂变方式 | 触发条件 | 奖励设计 | 适用品类 |
|---------|---------|---------|---------|
| **老带新** | 老客邀请新客首单 | 双方各得X元券 | 日用品/快消品 |
| **拼团** | 2-3人成团享折扣 | 团购价低于日常价15-30% | 标品/引流款 |
| **分销** | 用户分享链接成交 | 佣金比例5-20% | 高毛利品类 |
| **集赞/任务** | 分享到朋友圈集X赞 | 小额优惠券/赠品 | 需要社交传播的新品 |

#### 搜索端的老客运营

> 历史搜索记录是平台给老客的快速入口。用户的搜索记录里出现你的品牌词，说明已建立品牌认知。

| 阶段 | 动作 | 目标 |
|------|------|------|
| **首购后** | 包裹卡引导收藏店铺+关注 | 下次可从我的收藏直接进入 |
| **复购提醒** | 耗材/周期品到期前30天触达 | 引导通过搜索品牌词回店 |
| **品牌词竞价** | 对自己的品牌词做低价竞价防守 | 防止竞品截流老客搜索 |
> 核心原则：**私域不是另一个卖货渠道，而是用户关系资产。先给价值，再谈转化。**

## 难题六：新品试销评估（v3.0 新增）

> **定位**：线上电商新品从引进到评估的完整闭环，避免盲目铺货。

### 新品引进预估（规划阶段）

| 预估项 | 计算方式 | 数据来源 |
|--------|---------|---------|
| **目标销售量** | 竞品同价位中位数月销量 x 0.5（保守系数） | 竞品数据 |
| **目标销售额** | 目标销售量 x 定价 | 定价策略 |
| **目标在售店铺数** | 根据店铺矩阵确定 | 店铺矩阵 |
| **目标品类渗透率** | 新品目标购买客户数 / 总客户数 | 品类矩阵 |

### 试销控制（执行阶段）

**试销期定义**：新品上架后 14-30 天为试销期。

| 监控维度 | 指标 | 预警阈值 | 触发动作 |
|---------|------|---------|---------|
| **销量达成** | 实际销量 / 目标销量 | < 50%（连续7天） | 调整定价/标题/主图 |
| **库存周转** | 库存 / 日均销量 | > 60天 | 暂停补货 |
| **品类渗透** | 购买新品客户数 / 总客户数 | < 目标值50% | 加大推广/关联推荐 |
| **转化率** | 新品支付UV / 新品浏览UV | < 行业均值50% | 优化详情页/评价引导 |

### 新品贡献评估（监控阶段）

**试销期结束后（30天）输出评估报告**：

| 评估维度 | 指标 | 判断标准 |
|---------|------|---------|
| 消费者渗透率 | 购买新品客户数 / 总客户数 | > 5% 为合格 |
| 购买频次 | 新品购买次数 / 购买新品客户数 | 与店铺平均频次对比 |
| 新品利润率 | 新品毛利 / 新品销售额 | > 店铺平均毛利率 |
| 销售额份额 | 新品销售额 / 品类总销售额 | > 5% 为合格 |
| 品类牵引效应 | 新品上架后品类总销售额增幅 | > 0 说明新品带动品类增长 |
| 连带购买率 | 购买新品同时购买其他品的订单占比 | > 15% 说明有连带效应 |

### 新品决策矩阵

| 销量达成 | 利润率 | 品类牵引 | 决策 |
|---------|--------|---------|------|
| 达标 | 达标 | 正向 | **扩量**：增加备货，全店推广 |
| 达标 | 不达标 | 正向 | **优化成本**：谈判供应商或提价 |
| 不达标 | 达标 | 正向 | **加大推广**：优化标题/主图/投放 |
| 不达标 | 不达标 | 正向 | **观察**：再给15天试销期 |
| 任何情况 | 任何情况 | 负向 | **淘汰**：下架，清理库存 |

## A/B Test 模块（v3.0 新增，跨阶段决策工具）

> **定位**：当分析无法得出确定性结论时，用真实流量验证假设。
> **核心理念**：让市场数据投票，而不是让运营拍脑袋。

### 可测决策维度矩阵

| 决策层 | 测试变量 | 主要指标 | 次要指标 |
|--------|---------|---------|---------|
| **主图** | 白底图 vs 场景图 / 卖点排布 | CTR | 加购率 |
| **标题** | 核心词顺序 / 属性词组合 | CTR | 搜索排名 |
| **定价** | 价格A vs 价格B / 折扣vs券 | 转化率 | 客单价 |
| **详情首屏** | 痛点切入 vs 卖点切入 | 加购率 | 停留时长 |
| **促销形式** | 直降 vs 满减 vs 赠品 | 下单率 | 毛利率 |
| **SKU设计** | 规格合并 vs 拆分 | 动销率 | 客单价 |

### 标准执行五步流程

`
Step 1 — 定义假设（必须可证伪，同一测试只改一个变量）
Step 2 — 确定指标与最小样本（每组 >= 500曝光）
Step 3 — 执行测试（同期同人群，其他变量冻结）
Step 4 — 读数据做决策：
         差异 < 5%：统计不显著 -> 选成本更低的
         差异 5-15%：倾向胜出方，继续观察
         差异 > 15%：明确结论，全量切换
Step 5 — 归档结论（写入经验教训登记册）
`

### 各平台 A/B Test 工具

| 平台 | 工具名称 | 能测什么 |
|------|---------|---------|
| 淘宝/天猫 | 万相台ABTest / 直通车创意测试 | 主图、标题、详情 |
| 抖音 | 千川创意测试 | 视频素材、封面、文案 |
| 无平台工具时 | 双链接法 / 优惠券法 / 分时段轮换 | 价格、SKU设计 |

### 何时该测、何时不该测

| Should test | Should NOT test |
|---------|-----------|
| 两个方案各有道理，数据无法判断 | 样本量不足（<500曝光/组） |
| 涉及重大资源投入 | 大促前后（流量结构已变形） |
| 复盘发现方向模糊需校准 | 已有历史数据可参考 |

## Phase 3：监控（Monitoring & Controlling）

### 3.1 漏斗数据监控

```
每日 09:00 → 查看昨日漏斗数据
            → 识别最大流失环节（红标）
            → 判断：正常波动 / 需干预

每周一 → 漏斗周报：环比/同比分析
        → 识别趋势性恶化

每月初 → 漏斗复盘：归因分析
        → 更新SOP：什么情况下触发变更
```

### 3.2 EVM绩效指标

| 指标 | 公式 | 用途 |
|------|------|------|
| **CV（成本偏差）** | EV - AC | 预算执行情况 |
| **CPI（成本绩效）** | EV / AC | 花钱效率 |
| **CAC（获客成本）** | 总投放 ÷ 付费人数 | 流量效率 |
| **LTV（用户生命周期价值）** | 客单价 × 复购次数 | 用户价值 |

### 3.3 监控诊断协议

| 漏斗问题 | 诊断 | 变更请求 |
|---------|------|---------|
| 曝光→点击流失大 | 范围/质量偏差 | 优化主图、调整标题关键词 |
| 点击→激活流失大 | 详情页说服力不足 | 优化详情、增加买家秀 |
| 激活→下单流失大 | 价格/促销感知弱 | 调整定价、增加限时优惠 |
| 下单→付费流失大 | 支付信任度低 | 增加信任背书、简化支付流程 |

### 3.4 数据分析能力（盲区C4 新增 v2.0）

> ⚠️ 会看报表不等于会分析。真正的数据分析能力是：看到数据→定位问题→找到根因→给出可执行方案。

#### 数据分析五步法

```
Step 1：看趋势（同比/环比）
  → 这个数据是偶发波动还是持续下滑？
  → 工具：7日/30日趋势线

Step 2：做对比（自身 vs 行业 vs 竞品）
  → 这个数据在行业里是什么水平？
  → 工具：各平台行业均值对照

Step 3：拆维度（渠道/人群/时段/地域）
  → 是全局问题还是局部问题？
  → 工具：各维度数据下钻

Step 4：找关联（先导指标→结果指标）
  → 哪个先导指标出问题了？
  → 路径：曝光↓→点击↓→加购↓→成交↓

Step 5：出方案（定变量→定动作→定预期）
  → 改哪个变量？改多少？预期效果？
  → 输出：变更请求
```

#### 核心指标速查（含行业均值参考）

| 指标 | 健康范围 | 预警阈值 | 诊断方向 |
|------|---------|---------|---------|
| CTR（直通车） | 3-5% | <2% | 词/图/人群三者必有一错 |
| CTR（引力魔方） | 2-4% | <1% | 人群包不精准 |
| 加购率 | 8-15% | <5% | 详情页说服力不足 |
| 下单转化率 | 3-8% | <2% | 价格/促销感知弱 |
| 退货率 | 5-15% | >20% | 品质/描述不符/预期管理 |
| ROI（付费） | ≥2.5 | <1.5 | CAC过高或客单价太低 |

#### 数据健康度仪表盘模板

```
每日必看：
  □ GMV vs 昨日（环比）
  □ 付费花费 vs 预算 （超预算%）
  □ 主推款转化率（单日波动>30%需关注）
  □ 竞品排名变化（TOP3是否变动）

每周必看：
  □ 周GMV vs 目标（达成率%）
  □ 各渠道ROI（哪个渠道拖后腿）
  □ 库存周转天数（是否逼近预警线）
  □ 私域新增人数+活跃度

每月必看：
  □ 月度净利润（扣除所有成本后）
  □ 三层毛利率变动趋势
  □ 复购率+复购GMV占比
  □ 品牌搜索量变化
```

### 3.5 售后与口碑管理（盲区C5 新增 v2.0）

> ⚠️ 1个差评的影响超过100个好评。售后不是成本中心，是口碑资产管理部门。用好了售后，复购和转介绍自然来。

#### DSR（店铺评分）三维管理

| 维度 | 权重 | 核心动作 | 目标 |
|------|------|---------|------|
| 描述相符 | 高 | 实物与描述一致性控制 | ≥4.8 |
| 服务态度 | 中 | 响应速度+专业度+情感温度 | ≥4.8 |
| 物流速度 | 中 | 发货时效+物流追踪+异常处理 | ≥4.7 |

#### 售后分级响应机制

```
Level 1（常规问题，占80%）
  退换货/物流查询/使用咨询
  → 自动客服首轮响应（15秒内）
  → 无法解决自动转人工
  → 目标：2小时内解决

Level 2（中度问题，占15%）
  品质投诉/描述不符/漏发错发
  → 人工客服接入（5分钟内）
  → 补偿方案+解释+内部追责
  → 目标：24小时内闭环

Level 3（严重问题，占5%）
  安全投诉/批量质量/平台介入纠纷
  → 客服主管直管
  → 内部流程升级+全平台预警
  → 目标：2小时内出方案，48小时内彻底闭环
```

#### 差评处理标准流程

```
发现差评 → 5分钟内客服电话联系
  → 倾听（不解释、不推卸）→ 道歉 → 补偿方案
  → 内部追责：是产品问题？仓储问题？物流问题？描述问题？
  → 整改：更新描述/加强质检/更换物流/培训客服
  → 回访：3天后跟进用户是否满意
  → 归档：问题类型+解决方案+预防措施
```

#### 好评率提升四步法

| 步骤 | 动作 | 说明 |
|------|------|------|
| 超预期交付 | 实物比描述多20%价值感 | 好包装/小赠品/手写卡片 |
| 主动触达 | 签收后第3天关怀消息 | 「用得怎么样？有任何问题随时找我」 |
| 好评引导 | 满意的用户引导发好评 | 提供可选模板降低用户编辑成本 |
| 口碑放大 | 精选好评做成内容素材 | 详情页买家秀、社群口碑传播 |

> 核心原则：**差评用户是给了你改进机会的人。每一个差评背后都有系统性问题值得解决。**

### 3.6 品类健康度监控（v3.0 新增）

> **定位**：各品类是否按预设角色运转？品类结构是否健康？

#### 品类角色执行监控

| 品类角色 | 核心监控指标 | 预警阈值 | 触发动作 |
|---------|------------|---------|---------|
| **引流品类** | 品类渗透率、购买频次 | 渗透率 < 20% | 加大促销、优化标题 |
| **利润品类** | 毛利率、利润贡献率 | 利润贡献率 < 30% | 提价或控制成本 |
| **品牌锚点品类** | 独家产品占比、品牌词搜索占比 | 独家占比 < 20% | 开发独家款、强化差异化 |
| **便利品类** | 连带率、关联购买率 | 连带率 < 10% | 优化关联推荐、套餐设计 |

#### 品类VPE得分（综合健康度）

> **VPE 得分**：融合销量贡献 + 利润贡献 + 周转效率三维评价。

`
品类VPE得分 = 销量贡献得分 x 利润贡献得分 x 周转效率得分

销量贡献得分 = 该品类销售额 / 全店总销售额 x 100
利润贡献得分 = 该品类毛利额 / 全店总毛利额 x 100
周转效率得分 = 该品类库存周转天数 / 品类平均周转天数 x 100（越低越好，得分取倒数标准化）

健康阈值：
- > 80分：优秀（三维度均表现良好）
- 60-80分：良好（至少两个维度达标）
- 40-60分：预警（仅一个维度达标）
- < 40分：危险（三维度均不达标，考虑调整品类角色或清理SKU）
`

#### 品类SKU结构监控

| 指标 | 计算方式 | 健康阈值 | 异常信号 |
|------|---------|---------|---------|
| **品类SKU上限** | 根据品类角色和市场容量设定 | 引流品类可放宽 | 超出上限需清理 |
| **品类SKU下限** | 必备SKU数 + 基础覆盖数 | 必备SKU占比 > 80% | 低于下限需补充 |
| **品类动销率** | 有销量SKU / 品类总SKU | > 70% | < 50%需优化 |
| **品类新品占比** | 上架<90天的SKU数 / 品类总SKU数 | 10-20% | > 30%或 < 5% |

### 3.7 商品结构分析（v3.0 新增）

#### 二八分析（帕累托法则）

`
TOP20% SKU 贡献率 = TOP20% SKU 销售额 / 全店总销售额 x 100%

健康阈值：
- > 70%：健康（头部效应明显，爆款驱动力强）
- 50-70%：一般（腰部 SKU 有潜力，需优化）
- < 50%：危险（SKU 过于分散，无明确爆款）
`

#### 首选商品分析

| 指标 | 健康阈值 | 说明 |
|------|---------|------|
| 首选SKU占比 | > 60% | 首选 SKU 应覆盖品类核心需求 |
| 首选SKU动销率 | > 90% | 首选 SKU 必须动销 |
| 本店vs行业重合度 | > 50% | 重合度 < 50% 说明选品与市场需求偏差大 |

#### 滞销品分级处理

| 滞销等级 | 定义 | 处理策略 |
|---------|------|---------|
| **重度滞销** | 30天销量 = 0 | 立即下架，清仓处理 |
| **中度滞销** | 30天销量 1-3件 | 促销降价，捆绑销售 |
| **轻度滞销** | 30天销量 4-10件 | 优化标题/主图，观察15天 |

### 3.8 客户管理 RFM（v3.0 新增）

> **核心认知**：获取一个新客户的成本是维护一个老客户的 5-8 倍。

#### RFM客户分层模型

| 维度 | 含义 | 电商场景 |
|------|------|---------|
| **R**ecency（最近消费） | 最近一次购买时间 | 判断用户活跃度 |
| **F**requency（消费频率） | 购买次数 | 判断用户忠诚度 |
| **M**onetary（消费金额） | 累计消费金额 | 判断用户价值 |

#### 客户价值分层矩阵

| RFM组合 | 客户类型 | 运营策略 |
|---------|---------|---------|
| **高R高F高M** | 重要价值客户 | VIP维护、专属优惠、优先发货 |
| **高R低F高M** | 重要发展客户 | 提升购买频次、会员体系引导 |
| **低R高F高M** | 重要保持客户 | 召回激活、专属优惠刺激 |
| **低R低F高M** | 重要挽留客户 | 深度召回、问题调研 |
| **高R高F低M** | 潜力客户 | 升级引导、高毛利产品推荐 |
| **高R低F低M** | 新客户 | 培育激活、复购引导 |
| **低R高F低M** | 流失风险客户 | 召回优惠、流失预警 |
| **低R低F低M** | 一般客户 | 低成本触达、促销活动推送 |

#### 客户生命周期管理

| 阶段 | 核心动作 | 目标指标 |
|------|---------|---------|
| **获取期** | 新客首单优惠、引流款成交 | 新客转化率 |
| **培育期** | 使用引导、复购激励 | 30天复购率 |
| **成熟期** | VIP权益、专属推荐 | LTV、复购频次 |
| **衰退期** | 流失预警、召回激励 | 流失率、召回率 |
| **挽回期** | 问题调研、专属补偿 | 挽回率 |

## 多轮诊断协议（CTR/转化率问题时启用）

**CTR诊断标准追问链**：

```
Step 1（必问）："这个1.2%是哪个渠道的CTR？直通车 / 引力魔方 / 品牌特秀 / 搜索？"
Step 2（根据渠道）："对应的行业均值是多少——直通车均值3-5%，引力魔方2-4%，品牌特秀1-2%"
Step 3（缩小范围）："是词的问题（出价/匹配方式）还是人群包的问题？"
Step 4（给结论）：综合判断 → 归因 → 变更请求
```

**其他指标诊断追问链**：

| 指标 | 必问第一个问题 |
|------|--------------|
| 转化率低 | "是访客质量低还是详情页说服力不足？" |
| ROI低 | "是获客成本高还是客单价/复购率低？" |
| 收藏加购率高但下单率低 | "是否存在价格锚点问题，或促销感知弱？" |
| DSR下降 | "是哪个维度：描述/服务/物流？" |

## 复盘标准五问（阶段复盘时启用）

> **规则**：不直接给结论 → 按复盘标准问题集逐步追问 → 再出分析

```
Step 1（漏斗回顾）："哪个环节数据最差？是曝光/点击/加购/下单/付费？"
Step 2（归因分析）："这个环节的问题，原因更像是流量质量问题、详情页说服力问题、还是价格/促销感知问题？"
Step 3（策略调整）："针对这个原因，下个月优先改哪个变量？（主图/标题/定价/详情/投放策略）"
Step 4（资源复盘）："本月投放ROI和CAC是多少？有没有超预算的渠道？"
Step 5（经验归档）："这次复盘里，下次可以直接复用的经验是什么？"
```

## 错误处理表

| 错误场景 | 诊断 | 处理方式 |
|---------|------|---------|
| 数据低于行业均值 | 漏斗哪个节点异常 | 启动多轮诊断协议（见上） |
| 竞品分析数据缺失 | 来源受限 | 用 AI 推理 + 标注「估算」，建议用生意参谋验证 |
| 定价无法覆盖成本 | 毛利为负 | 重制定价方案，或调整产品结构 |
| 投放超预算 | CPI 或 CAC 过高 | 诊断归因 → 调整渠道/人群/出价 |
| 漏斗数据造假/不准 | 数据来源问题 | 核实数据源，排除刷单/异常流量 |
| SOP 效果不及预期 | 方案与环境不匹配 | 复盘五问 → 针对性调整 SOP |
| 产品资料不全就要上架 | 基础未打好 | 强制走难题一SOP，补齐再上架 |
| 投流ROI<1持续3天 | 产品/定价/页面问题 | 暂停投流 → 先修转化率 → 再重启 |
| 自动客服人工介入率>50% | 话术覆盖不足或问题太复杂 | 补充A类话术 + 重新梳理人工介入规则 |
| 上架后长期0曝光 | 收录异常或类目偏差 | 检查类目/属性/违规词 → 重新提交审核 |
| 账面盈利但现金断流 | 只算了账面毛利，忽略隐性成本+库存压款 | 启动三层毛利核算 → 库存健康体检 → 现金流预警 |
| 多店铺数据互斥冲突 | 库存/价格未同步 | 启动三同步机制（价格/库存/内容） |
| 私域沉淀率<10% | 引导机制缺失或福利不够吸引 | 检查包裹卡+客服话术落地情况，增强福利钩子 |
| 差评突然爆发 | DSR单维度快速下滑 | 启动差评处理标准流程 → 内部追责 → 全平台预警 |
| 数据分析结论相互矛盾 | 数据来源不一致或维度不同 | 统一数据口径和对比维度，核查五步法Step 3 |
| 问大家出现大量负面 | 产品质量或描述问题未解决 | 先解决产品问题 → 联系已购满意用户邀请回答 → 同步优化详情页 |
| 搜索排名持续下降 | 销量下滑/DSR降低/竞品上升 | 排查权重因子（销量/转化率/DSR/相关性）→ 逐一优化 |
| 包邮后转化无提升 | 运费不是核心决策因素 | 分析用户真实决策因素（价格/品牌/评价）→ 调整定价策略 |
| 裂变活动参与率低 | 奖励不够吸引或传播门槛太高 | 降低参与门槛+提高奖励价值+优化传播话术 |
